Stats_X

Advanced Virtual Machine Ability System for Unreal Engine 5

C++ UE5 Plugin FAB Marketplace Multiplayer Ready

Sistema professionale per la gestione di attributi e status effects, progettato come alternativa leggera e performante a GAS. ~75.000 linee di codice C++. Utilizza tecniche di bit-packing e BLOB architecture per massimizzare le prestazioni della CPU cache L1.

  • Attribute System — Health, Mana, Stats con auto-recalculation e delta compression
  • Modifier System — Buff/debuff stackabili con formula matematica e dipendenze customizabili
  • Interceptor System — Shields, lifesteal, damage reduction con ricerca O(1)
  • Status Forge — Editor node-based che compila in C++ native hot-path
  • Network Ready — Fast Array Replication, server-authoritative, zero config
  • Performance — 120 FPS con 500+ actors, 5000+ status concurrent

02

Inventory_X

Advanced Predictive Inventory System for Unreal Engine 5

C++17/20 UE5 Plugin FAB Marketplace MMO Scale

Plugin professionale per la gestione inventari con architettura lag-compensated in stile sparatutto competitivo. Progettato per scala MMO con zero allocazioni heap nelle operazioni ad alta frequenza e sistema di sicurezza anti-cheat integrato.

  • Client-Side Prediction — Il client predice le mutazioni (consumo munizioni, spostamento item) e il server riconcilia con rollback automatico
  • Rollback Journaling — Shadow State con sistema undo/redo interno: desincronizzazione visiva gestita senza UI flickering
  • FastArray Replication — Delta serialization MMO-scale: solo i dati modificati viaggiano in rete
  • Anti-Cheat Security — Custom NetSerialize con Payload Bounding: previene Allocation Bomb scartando payload malevoli prima dell'heap allocation
  • Zero-Allocation Hot Paths — MicroSnapshots stack-allocated (<64 byte) e TInlineAllocator: 0 malloc nelle operazioni a 60Hz+
  • Data-Driven Design — Policy Pattern su UDataAsset: regole di validazione e mutazione modificabili senza toccare l'hot-path C++
  • Custom K2 Nodes — Blueprint nodes con wildcard pins, type-checking a compile-time e Custom Thunks che bypassano la Blueprint VM
