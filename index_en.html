Stats_X

Advanced Virtual Machine Ability System for Unreal Engine 5

C++ UE5 Plugin FAB Marketplace Multiplayer Ready

Professional system for managing attributes and status effects, designed as a lightweight and performant alternative to GAS. ~75,000 lines of C++. Uses bit-packing and BLOB architecture techniques to maximize CPU cache L1 performance.

  • Attribute System — Health, Mana, Stats with auto-recalculation and delta compression
  • Modifier System — Stackable buff/debuff with mathematical formula and customizable dependencies
  • Interceptor System — Shields, lifesteal, damage reduction with O(1) lookup
  • Status Forge — Node-based editor that compiles to native C++ hot-path
  • Network Ready — Fast Array Replication, server-authoritative, zero config
  • Performance — 120 FPS with 500+ actors, 5000+ concurrent status effects

Inventory_X

Advanced Predictive Inventory System for Unreal Engine 5

C++17/20 UE5 Plugin FAB Marketplace MMO Scale

Professional inventory management plugin, completely UI Agnostic and genre-independent. It imposes no concept or visual structure: it provides all the tools to build any type of inventory. Lag-compensated architecture, zero heap allocations on high-frequency operations, and an advanced security system covering exploits, network data manipulation, and much more.

  • Client-Side Prediction — Client predicts mutations (ammo consumption, item movement) and the server reconciles with automatic rollback
  • Rollback Journaling — Shadow State with internal undo/redo system: visual desync handled without UI flickering
  • FastArray Replication — MMO-scale delta serialization: only modified data travels over the network
  • Anti-Cheat Security — Custom NetSerialize with Payload Bounding: prevents Allocation Bombs by discarding malicious payloads before heap allocation
  • Zero-Allocation Hot Paths — Stack-allocated MicroSnapshots (<64 bytes) and TInlineAllocator: 0 malloc at 60Hz+
  • Data-Driven Design — Policy Pattern on UDataAsset: validation and mutation rules modifiable without touching the C++ hot-path
  • Custom K2 Nodes — Blueprint nodes with wildcard pins, compile-time type-checking and Custom Thunks that bypass the Blueprint VM
